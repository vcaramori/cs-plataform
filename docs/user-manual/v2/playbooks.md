# Manual do Usuário — Playbooks & Automação (Avaliação 2.0)

Bem-vindo ao manual da tela de **Playbooks & Automação**. Esta é a área onde definimos os processos padronizados de Customer Success.

---

## 🎯 O que esta tela faz?

Esta tela lista os templates de Playbooks disponíveis. Playbooks são sequências de tarefas que devem ser executadas em situações específicas (ex: "Risco de Churn", "Onboarding").

---

## 📊 Regras de Negócio e Memória de Cálculo

### 1. Gatilhos Automáticos (Triggers)
- **O que são**: Regras que disparam a criação de um playbook para uma conta automaticamente.
- **Exemplo**: Se o Health Score de uma conta cair abaixo de 50, o playbook de "Recuperação de Conta" é ativado.
- **Configuração**: A condição de gatilho aparece no card de cada playbook.

### 2. Etapas (Tasks)
- Cada playbook é composto por uma ou mais etapas que o CSM deve cumprir.
- O número de etapas configuradas aparece no rodapé do card.

---

## 💡 Como Usar a Tela

1. **Visualizar Playbooks**: Você vê os cards com os playbooks ativos e inativos.
2. **Novo Playbook**: Botão no topo para criar um novo processo (Redireciona para o Builder).
3. **Builder (Em Desenvolvimento)**: Uma área visual onde você poderá arrastar blocos para criar regras (atualmente em fase de atualização).
